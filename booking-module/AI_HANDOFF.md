# 🤖 Документ для передачи проекта (AI Agent Handoff)

Добро пожаловать в систему бронирования клиники **Habbullo-Hilola**. Если вы — новый ИИ-агент, назначенный на этот проект, начните отсюда. Этот документ содержит ключевой контекст, который поможет вам понять кодовую базу без необходимости читать каждый отдельный файл.

## 📌 Обзор проекта
Это современное веб-приложение для медицинской клиники (стоматология, ЛОР и т.д.), которое обрабатывает записи пациентов, управляет графиком врачей, ведет учет инвентаря/закупок, а также рассылает уведомления через Telegram бота.

## 🛠️ Технологический стек
- **Фреймворк:** Next.js 14/15 (App Router используется для API, а директория `public/` для статического фронтенда на HTML).
- **База данных:** PostgreSQL (развернута на Neon.tech), управляется через **Prisma ORM**.
- **Стилизация:** Чистый CSS (Vanilla CSS) в статических HTML-файлах (современные градиенты, глассморфизм). Для фронтенда НЕ используются React компоненты.
- **Уведомления:** Telegram Bot API (`node-telegram-bot-api`) для отправки сообщений пациентам и врачам.

## 📂 Основная архитектура
- `public/`: Содержит все фронтенд файлы (HTML/JS/CSS).
  - `index.html`: Главная страница и форма записи для пациентов.
  - `admin.html`: Главная панель для роли `ADMIN`.
  - `inventory.html`: Панель для роли `INVENTORY` (Ta'minotchi / Снабженец) для управления закупками.
- `src/app/api/`: API эндпоинты Next.js App Router.
  - `/api/auth`: Авторизация (JWT).
  - `/api/admin/`: Эндпоинты админа (врачи, пациенты, приемы, статистика).
  - `/api/bot/`: Webhook для приема запросов от Telegram бота.
- `prisma/schema.prisma`: Схема базы данных. Ключевые модели: `User`, `Doctor`, `Patient`, `Appointment`, `Procedure`, `Leave`, `Purchase`.
- `src/lib/`: Вспомогательные утилиты (`prisma.ts`, `auth.ts`, `telegram.ts`).

## ⚠️ Важные правила для агентов
1. **Изменения фронтенда:** НЕ ищите React компоненты (`.tsx`) для пользовательского интерфейса. Весь интерфейс написан на чистом HTML/JS внутри папки `public/`.
2. **Изменения базы данных:** Если вы меняете `prisma/schema.prisma`, вы ОБЯЗАНЫ выполнить `npx prisma db push`, чтобы синхронизировать изменения с базой Neon PostgreSQL. Не используйте `prisma migrate dev`, если вас об этом специально не попросили.
3. **Переменные окружения:** Файл `.env` содержит критически важные секреты (Neon DB URL, токены Telegram). Не меняйте их без явного указания пользователя.
4. **Использование инструментов:** Всегда используйте `grep_search` и `view_file` для поиска информации. Избегайте команд в bash вроде `cat`, `grep` или `ls`.

## 📝 Как логировать вашу работу (Решение проблемы с длинными файлами)
Чтобы агентам не приходилось со временем читать огромные логи, мы используем **систему скользящего журнала (Rolling Changelog)**. 
- Каждый раз, когда вы завершаете задачу, записывайте ее в `AI_CHANGELOG.md`.
- **КРИТИЧЕСКИ ВАЖНО:** Всегда добавляйте свои записи в **САМЫЙ ВЕРХ** файла `AI_CHANGELOG.md`.
- **ЧТЕНИЕ ЛОГОВ:** Когда вы только присоединяетесь к проекту, **читайте только первые 50 строк** из `AI_CHANGELOG.md`. Вам не нужно читать весь файл, если только вы не ищете информацию о каком-то старом баге. Если файл превышает 300 строк, перенесите нижние 200 строк в новый архивный файл `AI_CHANGELOG_ARCHIVE.md`.
